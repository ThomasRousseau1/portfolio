Projets

Site personnel

Création d'un site vitrine pour le lancement de mon activité de professeur de guitare. Le site est responsive et présente quelques modifications en version mobile par rapport à la version desktop.

Groupomania

L'objectif de ce projet était de créer un réseau social d'entreprise visant à faciliter les interactions entre les collègues de Groupomania. Pour cela, j'ai décidé d'utiliser la bibliothèque Javascript React pour gérer le frontend.

L'API a été crée grâce au framework node Express et utilise divers plugins pour sécuriser le site comme bcrypt, jsonwebtoken, password-validator, cors, dotenv ou encore multer pour les images.
J'ai décidé d'utiliser Sequelize pour intéragir avec la base de données MySql en passant par PhpMyAdmmin pour l'interface.

Hot Takes

Sixième projet de la formation Développeur Web d'OpenClassrooms. Le but était de réaliser le backend de manière sécurisée en partant de 0 et en ayant uniquement le frontend à disposition.

Pour la base de données j'ai choisi MongoDB pour y stocker les informations de connexion des utilisateurs ainsi que les informations concernant les sauces ajoutées par ces derniers. Au niveau de la sécurité, j'ai utilisé divers plugins comme JSON Web Token, bcrypt, multer, password-validator.

J'ai stocké l'url MongoDB contenant les informations de connexion à la base de données via une variable d'environnement stockée dans un fichier .env, tout comme le Token.

Oricam

Projet n°5 et premier projet où je devais utiliser du Javascript pur dans le cadre de la formation d'OpenClassrooms. L'objectif était de réaliser le frontend d'un site de e-commerce avec le backend à disposition.

Pour la mise en forme du code, j'ai choisi d'utiliser le pattern MVC (Model, Vue, Controller) pour une meilleure organisation du code. J'ai utilisé fetch pour afficher les données provenant de l'API et le localstorage pour la persistance des données, notament pour l'ajout d'un produit au panier.

La Chouette Agence

Pour ce projet, le but était d'optimiser les performances, la SEO ainsi que l'accessibilité d'un site complet.
Concernant la SEO, certaines anciennes techniques de black hat étaient utilisées sur le site : divs remplies de keywords cachées, beaucoup de liens cassés présents sur le footer, textes sous forme d'images...etc.

Au niveau de l'accessibilité, les textes étaient trop petits et trop clairs pour certains. Le respect des normes WCAG avait était omis. Pour ce qui est des performances, les images avaient une résolution beaucoup trop élevées par rapport à leur affichage sur page. Encore plus pour la version mobile du site.

Voici le site avant optimisation

OhMyFood

3ème projet de la formation d'OpenClassrooms. Il fallait intégrer la maquette de la startup Ohmyfood de manière responsive et en utilisant SASS pour le CSS. Un cahier des charges était fourni avec toutes les informations nécessaire pour respecter au mieux la maquette fournie.

L'objectif principal était de réaliser des effets : remplissage progressif du coeur sur les cartes de chaques restaurants sur la page d'accueil, affichage dynamique des menus en arrivant sur les pages de chacuns des restaurants par exemple.

Reservia

Second projet de la formation Développeur Web d'OpenClassrooms. L'objectif était d'intégrer la maquette de Reservia, une agence de réservation de vacances. Le site devait être totalement responsive et respecter les préconisations en terme de couleurs fournies, icônes et polices.
